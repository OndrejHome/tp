Below table shows state of reposync for repositories on local server.

  • == repocheck == - checking if we have access to repository before starting sync
  • OK repo access - we can access repository with current credentials but the sync has not yet started
  • ERROR no repo access - we cannot access repository with current credentials, sync cannot start
  • == syncing == - repository is currently syncing, you may experience errors with missing packages until the sync is finished.
  • OK - repository was succesfully synchronized, check 'Last change' for date of sync.
  • SYNC ERROR - there was error during reposync, repository could be damaged or incomplete.
